    04_ExecutionEngine - Index¶

    这份 Index 的用法（面向新同学）¶

    把它当“学习路线图”用： - 30 分钟：建立执行引擎整体模型（看图为主） - 2 小时：能定位常见问题（按场景选读） - 1 天：能读懂实现并改代码（下潜逐行）

    建议同时打开： - README（主线图 + 3 场景排障树） - FileNotes/_Glossary（术语速查） - Newbie_MinDebug_Playbook（新人最小调试手册：确认 interpreter-type/fast interpreter/bridge/OSR）

    30min 路线：建立整体模型（不读逐行）¶

    1. Flows/ExecutionEngine_EndToEnd（端到端脊柱图：先把整条链路串起来）
    2. README（3 场景排障树 + 关键现实差异）
    3. Flows/Index（知道有哪些主调用链，每条链路的入口/出口）
    4. DataStructures/Index（Frame/VReg/Acc/Entrypoint/StackWalker/Deopt/OSR 的卡片）
    5. （真实场景补充）Flows/IRTOC_FastInterpreter + DataStructures/IRTOC_FastInterpreter（理解默认的 llvm/irtoc fast interpreter）
    6. 如果你需要“真正改 .irt”：Flows/IRTOC_DSL_Primer（全链路分工 + 怎么改/怎么验）
    7. 想更系统学 DSL：Flows/IRTOC_DSL_Reference（查手册式参考）
    8. （新人强烈建议）Flows/Opcode_DeepDives_IRTOC + DataStructures/ISA_and_OpcodeModel（从常用 opcode 建立直觉）

    2h 路线：按你遇到的问题选读¶

    场景 A：解释器执行异常/语义不对¶

    • Flows/Interpreter_Execute（解释器执行主链路：PC/Frame/Acc/VReg 读写点）
    • DataStructures/Frame_VReg_Acc（Frame/VReg/Acc 的语义与不变量）
    • 若你跑的是默认配置（llvm）：Flows/IRTOC_FastInterpreter（确认实际执行路径/dispatch table/handler 来源）
    • 逐行证据入口：FileNotes/Index（需要时再下潜）

    场景 B：I2C/C2I 桥接相关问题¶

    • Flows/Bridge_I2C_C2I（桥接的入口、frame kind 切换、返回值语义）
    • FileNotes/runtime_bridge_bridge.cpp（InvokeInterpreter 与桥接骨架）
    • （需要“落到栈/ABI/寄存器”的证据时）见 Flows/Bridge_I2C_C2I 的 4.1 arch 汇编证据链：
    • aarch64/amd64：I2C/C2I/proxy/deopt（含 dyn 版本入口）

    场景 C：deopt/OSR/stack walking¶

    • Flows/Deopt_and_OSR（deopt/OSR 与桥接/解释器的关系）
    • FileNotes/runtime_deoptimization.cpp、FileNotes/runtime_osr.cpp
    • FileNotes/runtime_include_stack_walker.h、FileNotes/runtime_stack_walker.cpp
    • （deopt-after 的最终落地）runtime/bridge/arch/*/deoptimization_*.S（对应 FileNotes 已挂在 Flows/Bridge_I2C_C2I 的 4.1）

    场景 D：编译器接口/entrypoints（compiled code 的慢路径）¶

    • Flows/Entrypoints_and_RuntimeInterface
    • FileNotes/runtime_entrypoints_entrypoints.cpp
    • FileNotes/runtime_compiler.cpp
